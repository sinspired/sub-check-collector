# V2Ray 订阅链接自动收集与更新器 - 完整功能说明

## 🎯 核心功能

本程序实现了从 GitHub 自动搜索、收集 V2Ray/Clash 订阅链接,并自动更新到 `config.yaml` 配置文件的完整流程。

## 📋 工作流程

```
GitHub 搜索 (3 策略并行) → 合并去重 → 时间验证 → 排序截断
                                                      ↓
                                              处理 50 个仓库 (5 并发)
                                                      ↓
                                              README 解析 → 链接提取
                                                      ↓
                                                双重输出
                                                  ↓     ↓
                                        Markdown 文件  config.yaml
```

## 🔄 详细步骤

### 1. 搜索阶段

- 3 种策略并行搜索: 关键词轮询、Code Search、Topics 搜索
- 全量收集所有发现的仓库（不早期截断）
- 按 `updated_at` 排序，取前 N*5 个候选
- 用 GitHub API 验证实际最后提交时间
- 最终取最新的 `MAX_REPOSITORIES` 个仓库

### 2. 解析阶段

- 5 并发处理仓库，大幅提升速度
- 自动读取每个仓库的 README 文件
- 使用多种正则表达式匹配订阅链接
- 智能识别链接类型(V2Ray/Clash/Shadowsocks/Hysteria/TUIC/WireGuard)

### 3. 处理阶段

- O(n) 高效去重（baseUrlMap）
- 按类型分类整理
- 记录来源和发现时间

### 4. 输出阶段

#### 输出 1: Markdown 文件 (`output/subscriptions.md`)

- 📊 统计信息(总数、分类统计)
- 🗂️ 按类型分组的详细链接
- 📝 每个链接的元数据(来源、描述、时间)
- 📎 纯链接列表(方便复制)

#### 输出 2: config.yaml 自动更新

- 💾 **自动备份**: 更新前自动备份原文件(带时间戳)
- 🔄 **智能合并**: 保留原有链接,添加新链接
- 📝 **保留注释**: 不破坏 YAML 文件的注释和格式
- ✅ **去重处理**: 自动过滤重复链接

## 🎨 特色功能

### 1. 智能链接过滤

只保留有效的订阅链接:

- `raw.githubusercontent.com` 链接
- `gist.githubusercontent.com` 链接
- 包含 `.txt`, `.yaml`, `.yml` 等订阅格式的链接
- 包含 `sub`, `subscription` 关键字的链接

自动过滤:

- 目录链接（以 `/` 结尾）
- LICENSE、README、CHANGELOG 文件
- 图片、徽章、QR 码
- 广告过滤列表、GKD 规则
- 编程语言/框架项目

### 2. 类型智能识别

根据上下文和 URL 路径自动识别:

- **V2Ray**: 包含 v2ray, vmess, vless, trojan, xray 关键字
- **Clash**: 包含 clash, mihomo 关键字
- **Shadowsocks**: 包含 shadowsocks, ss, ssr 关键字
- **Hysteria**: 包含 hysteria, hy2 关键字
- **TUIC**: 包含 tuic 关键字
- **WireGuard**: 包含 wireguard, wg 关键字
- **订阅链接**: 包含订阅、subscription、sub 关键字

### 3. 高效去重

- O(n) 复杂度的 baseUrlMap 去重
- 同一文件不同格式（如 all.yaml 和 v2ray.txt）只保留一个
- URL 规范化后比较，避免重复

### 4. 并发处理

- 仓库处理 5 并发，大幅提升速度
- 搜索策略并行执行
- 链接验证并发处理

## ⚙️ 配置选项

### 环境变量配置 (`.env`)

```env
# GitHub Token (可选,但推荐)
GITHUB_TOKEN=your_github_token_here

# 搜索关键字 (逗号分隔)
SEARCH_KEYWORDS=free,v2ray,subscription,vless

# 定时执行规则 (cron 表达式)
SCHEDULE_INTERVAL=0 2 * * *

# Markdown 输出文件路径
OUTPUT_FILE=./output/subscriptions.md

# 最大搜索仓库数
MAX_REPOSITORIES=50

# config.yaml 文件路径 (留空则不更新 YAML)
CONFIG_YAML_PATH=./config.yaml

# 仓库最大更新天数
MAX_DAYS_SINCE_UPDATE=60

# 代理地址
PROXY_URL=http://127.0.0.1:7890

# 订阅文件最大更新天数
MAX_DAYS_SINCE_SUB_UPDATE=3

# 是否验证链接有效性
VALIDATE_LINKS=true
```

## 📈 性能特点

- ⚡ **高效**: 5 并发处理,约 200 秒完成 50 个仓库
- 🔒 **安全**: 自动备份,不会丢失数据
- 💾 **节省**: O(n) 去重,避免重复处理
- 🎯 **精准**: 多种过滤规则,确保链接质量

## 🛡️ 安全措施

1. **自动备份**: 每次更新前自动备份配置文件
2. **错误处理**: 遇到错误不中断整体流程
3. **速率控制**: 自动检测 GitHub API 限制并等待重试
4. **格式保护**: 不破坏 YAML 文件的原有格式和注释

## 🔧 故障排查

### 问题: GitHub API 限制

**解决**: 配置 `GITHUB_TOKEN` 环境变量

### 问题: 不想更新 config.yaml

**解决**: 在 `.env` 中设置 `CONFIG_YAML_PATH=`

### 问题: 需要恢复旧配置

**解决**: 使用备份文件 `config.yaml.backup.*`
